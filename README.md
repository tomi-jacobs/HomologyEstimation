Homology estimation Homology estimation was conducted following the procedure developed by Yang and Smith (2014) and further refined by Morales-Briones et al. (2021). Each inferred transcriptome was first compressed to remove redundancy using CD-HIT v.4.8.1 with parameters -c 0.99 -n 10 -r 0. An all-by-all similarity search was conducted using BLASTN, with the setting “--evalue 10 –max_target_seqs 1000” on all transcriptome assemblies to infer putative homolog groups, which were then refined using Markov Clustering, with an inflation value of 1.4, which was selected as a moderate clustering stringency, balancing cluster fragmentation and over-merging, as implemented in mcl v14-137. The resulting homolog clusters were then used as input for the subsequent tree-based homology refinement. Tree based homolog refinement was then conducted on each resulting putative homolog by first aligning using MAFFT v7.490 with the settings “--auto --maxiterate 1000”, the resulting alignments were then cleaned for a minimum of 10% column occupancy using the Phyx v.1.2 program pxclsq. Phylogenetic trees were inferred using maximum-likelihood as implemented in IQ-TREE2 v2.3.6 (Minh et al., 2020) under the GTR+G model of evolution. Any branches on the resulting homolog trees that had a relative value of 0.1 subs/site or an absolute value of 0.2 subs/site were removed. Any clade that consisted of only the same taxa was then masked, so only the tip with the most character-rich alignment remained. Finally, any clade that contained at least 10 taxa and was on a branch of 0.2 subs/site or greater was then split into a separate homolog. All resulting homologs were then evaluated to determine whether the now tree based refined homolog had changed in the number of sequences. All homologs that changed in sequence number were then subjected to the tree refinement procedure until no further changes occurred. Once no further changes were detected, final homolog trees were rerun with 1000 ultra bootstrap replicates, and orthologs were extracted using Maximum Inclusion (MI) with relative and absolute branch length cutoffs of 0.1 and 0.2 sub/site, respectively, and a minimum taxon occupancy of 20 taxa.
